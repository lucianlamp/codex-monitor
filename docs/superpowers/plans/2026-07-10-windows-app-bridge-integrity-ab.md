# Windows Codex App Bridge Integrity A/B Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Determine whether an explicit `CODEX_CLI_PATH`, an unsigned executable, or its directory causes Codex Browser Use to reject ordinary navigation on Windows.

**Architecture:** Run two restart-gated variants against the same Codex App task. Variant A points directly to the OpenAI-signed App-managed CLI; Variant B places the unchanged unsigned monitor bridge beside that signed CLI and launches the signed sibling as its child. Record process, signature, monitor, and Browser evidence, then restore a known-safe native state before implementing the winning architecture.

**Tech Stack:** PowerShell 7, Windows Authenticode, Codex App in-app Browser Playwright surface, Rust `cdxm` diagnostics, Codex App user environment variables.

- [ ] **Step 1: Record hashes and signatures**

Run:

```powershell
$native = 'C:\Users\ytvar\AppData\Local\OpenAI\Codex\bin\a7c12ebff69fb123\codex.exe'
$bridge = "$HOME\.codex-monitor\bin\cdxm-codex-app-bridge.exe"
foreach ($path in @($native, $bridge)) {
    $signature = Get-AuthenticodeSignature -LiteralPath $path
    [pscustomobject]@{
        Path = $path
        SHA256 = (Get-FileHash -LiteralPath $path -Algorithm SHA256).Hash
        Signature = $signature.Status
        Signer = $signature.SignerCertificate.Subject
    }
}
```

Expected: native SHA-256 `B88F944EF63556527CAAE2AD43F80B88B8BE174DC09B09D9B037FC94240A0E91` with `Valid` OpenAI signature; bridge SHA-256 `CCFC541ED9F22F6BD5734DBC3138C4B0B7A0836AED4E0639C49DB89D8955317B` with `NotSigned`.

### Task 4: Configure Variant B — unsigned bridge beside signed native CLI

Run this task only when Variant A passed.

**Files:**
- Create external file: `C:\Users\ytvar\AppData\Local\OpenAI\Codex\bin\a7c12ebff69fb123\cdxm-codex-app-bridge.exe`

- [ ] **Step 1: Stage the owned bridge without replacing OpenAI files**

Run:

```powershell
$runtime = 'C:\Users\ytvar\AppData\Local\OpenAI\Codex\bin\a7c12ebff69fb123'
$source = "$HOME\.codex-monitor\bin\cdxm-codex-app-bridge.exe"
$staged = Join-Path $runtime 'cdxm-codex-app-bridge.exe'
Copy-Item -LiteralPath $source -Destination $staged -Force
if ((Get-FileHash -LiteralPath $source -Algorithm SHA256).Hash -ne
    (Get-FileHash -LiteralPath $staged -Algorithm SHA256).Hash) {
    throw 'staged bridge hash mismatch'
}
Get-AuthenticodeSignature -LiteralPath $staged |
    Select-Object Path, Status
```

Expected: only the new codex-monitor-owned filename is created; hash is unchanged and signature status is `NotSigned`.
