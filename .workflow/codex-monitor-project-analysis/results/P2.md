# P2 Implementation Map Result

Runtime flow:

1. CLI parses global target/endpoint options and command.
2. Target resolver returns managed, app, or explicit endpoint.
3. Transport opens ws, stdio, Unix app socket, or managed app-server child.
4. AppServerClient performs initialize/initialized handshake.
5. Commands call thread/list, turn/start, or agmsg watch delivery.

Main modules:

- `cli`: command surface and dispatch.
- `protocol`: JSON-RPC shaped builders and incoming classifier.
- `client`: request IDs, pending response cleanup, initialize, list, read, send.
- `transport`: trait plus memory, ws, stdio, Unix implementations.
- `target`: endpoint selection and thread/list response parsing.
- `sources::agmsg`: direct SQLite polling adapter.
- `delivery`: event formatting and foreground agmsg watch loop.
- `state`: JSON last-seen persistence.

Notable contract deviations or sharp edges:

- Managed WebSocket readiness checks TCP connect, not the documented `/readyz`
  HTTP response.
- State persistence tracks last seen agmsg ID per recipient key, not a general
  delivered event-id registry.
- No lock-file/concurrent writer rejection exists in `state`.
- `threads` output is id/title/cwd only, while the spec asks for richer
  metadata when available.
