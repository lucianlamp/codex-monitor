# P3 Test Contract Result

Automated tests cover:

- Binary and clientInfo naming contract.
- Protocol builders for initialize, initialized, thread/list, thread/read,
  turn/start, and explicit thread/inject_items.
- Client request/response lifecycle, terminal turn completion, and server
  request refusal.
- Target option precedence and current app-server `data` thread/list shape.
- agmsg SQLite fixture polling and formatting.
- Fake WebSocket app-server integration for `threads` and `send`.
- Loopback WebSocket endpoint refusal for non-loopback hosts.
- JSON state load/save/overwrite behavior.

Unproven or only smoke-tested:

- Live App daemon attach through `--target app`.
- Live mutating `send` into a real Codex thread.
- Long-running agmsg watch under real concurrent message arrival.
- Duplicate delivery behavior when two watch processes share one state file.
